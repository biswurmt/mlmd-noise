\subsection{Label imputation performance}

Table~\ref{tab:imputation_results} compares the two imputation strategies against the regex baseline across all four diagnostic pathways.

KG String Matching showed extreme variance across pathways. It performed well on Appendix Ultrasounds ($F_1 = 0.801$), where clinical documentation is standardized enough for lexical rules to succeed, but failed on Arm X-Rays (Recall: 0.031), where arm injury documentation varies too widely for substring matching. On Testicular Ultrasounds, precision collapsed to 0.048 (20,035 discoveries) because anatomical terms in that pathway overlap with unrelated symptom descriptions that a substring match cannot disambiguate.

GraphRAG Semantic Retrieval was substantially more consistent. Operating over semantic similarity rather than lexical overlap, it achieved 0.944 precision on Testicular Ultrasounds and 0.733 recall on Arm X-Rays, recovering the presentations that regex and string matching missed. The exception was Appendix Ultrasounds, where recall dropped to 0.198, likely because the embedding space flattens distinctions that lexical rules preserve in this narrowly documented pathway. Across all four pathways, GraphRAG identified 11,110 total discoveries. Both imputation strategies operate within acceptable bounds for offline retrospective use. String matching requires negligible overhead (0.55 ms per encounter, 6.16 ms cold-start initialization). GraphRAG semantic retrieval incurs higher cost due to the local biomedical embedding model (175.54 ms per encounter, 17.75 s initialization), but this latency is clinically inconsequential for offline imputation and functionally instantaneous for interactive query during prospective clinical use.
