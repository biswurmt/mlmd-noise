We compare four training conditions: (1) \textit{Production Baseline}, a standard classifier trained on the original regex-imputed labels; (2) \textit{GraphRAG Imputation Only}, retrained on the KG-imputed dataset with standard training; (3) \textit{LiLAW Only}, trained on the original regex labels with LiLAW-weighted training; and (4) \textit{Combined}, LiLAW-weighted training on the KG-imputed dataset.

Recall (true positive rate) is the primary metric. It measures the fraction of needed tests the model authorizes at triage, directly quantifying efficiency: each authorized test completes while the patient waits for assessment, while each miss returns the patient to the serialized queue. Precision (positive predictive value) is the safety constraint, ensuring the system does not burden ancillary departments with unwarranted test authorizations. We report precision-recall (PR) curves at multiple operating thresholds rather than single-threshold summaries, because clinical deployment requires understanding the full tradeoff. AUROC captures overall discriminative ability under severe class imbalance. Expected Calibration Error (ECE) measures whether model confidence scores can be interpreted as meaningful probabilities, which determines how much weight clinicians can place on the system's predictions.

The imputation pipeline is evaluated separately from the downstream model. The production regex system serves as a high-precision, low-recall baseline. Novel positive imputations absent from the regex baseline are classified as ``discoveries'': potential clinical false negatives the regex missed.
